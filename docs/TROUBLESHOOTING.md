# Диагностика

## Нет текущей температуры

Проверьте:

- RX и TX не перепутаны;
- общую землю;
- UART `9600 8E1`;
- наличие полного входящего кадра;
- правильность уровня на RX ESP32.

Рабочий кадр этой ревизии начинается с:

```
BB 01 00 04 3B
```

Значение `0x3B` означает тело длиной 59 байт, поэтому полный пакет занимает 65 байт.

## Invalid checksum

Частая причина — использование компонента, ожидающего пакет длиной 61 байт.

Этот компонент рассчитан на 65 байт.

## После команды включается неправильный режим

Проверьте, что используется актуальная версия компонента. В ней сопоставление режимов:

```
COOL     -> 0x03
FAN_ONLY -> 0x07
DRY      -> 0x02
HEAT     -> 0x01
AUTO     -> 0x08
```

## Большая задержка кнопок

Управляющие команды отправляются немедленно. Параметр:

```
update_interval: 3s
```

определяет период опроса состояния, а не задержку отправки команды.

Не ставьте интервал 1 мс: UART 9600 бод физически не сможет обработать такой поток запросов и ответов.

## Не работает штатный ИК-пульт

Проверьте:

- не перегружена ли UART-линия слишком частым опросом;
- нет ли ошибок согласования уровней;
- не отправляет ли ESP повторяющиеся управляющие команды;
- используется ли `update_interval: 3s`.

## Переключатель звука выключен, но пульт всё равно пищит

Переключатель управляет командами, которые отправляет ESP32. ИК-пульт может формировать собственные команды со звуковым подтверждением.

## Дисплей снова включился

Штатный пульт или другая система управления может повторно изменить состояние дисплея.

## Wi-Fi не подключается

Публичная конфигурация использует DHCP. Проверьте:

- `secrets.yaml`;
- SSID и пароль;
- наличие сети 2,4 ГГц;
- резервную точку доступа `${friendly_name} Recovery`;
- журнал загрузки по USB.
