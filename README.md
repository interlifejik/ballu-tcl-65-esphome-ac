# Ballu/TCL ESPHome — протокол с ответом 65 байт

Локальный внешний компонент ESPHome для некоторых кондиционеров на платформе TCL, которые отвечают по UART пакетом длиной **65 байт**, а не стандартным пакетом длиной 61 байт.

Проект подготовлен и проверялся на:

- Ballu BSVI-09HN8 Discovery;
- ESP32-C3 DevKitM-1;
- UART `9600 8E1`;
- ESPHome 2026.5.1;
- Home Assistant.

> Совместимость с другой моделью, ревизией платы или прошивкой кондиционера не гарантируется. Даже одинаковые торговые модели могут иметь разные контроллеры.

## Возможности

- включение и выключение;
- режимы `AUTO`, `COOL`, `DRY`, `FAN_ONLY`, `HEAT`;
- текущая и целевая температура;
- скорости вентилятора;
- стандартное вертикальное, горизонтальное и одновременное качание;
- вертикальное качание только в верхней или нижней части;
- горизонтальное качание в левой, центральной или правой части;
- точная фиксация жалюзи по позициям;
- отключение звукового подтверждения команд;
- включение и выключение дисплея внутреннего блока;
- немедленная отправка управляющих команд;
- опрос состояния раз в 3 секунды;
- поддержка входящего ответа длиной 65 байт.

## Почему отдельная версия

Некоторые реализации TCL ожидают ответ:

```
5 байт заголовка + 55 байт тела + 1 байт контрольной суммы = 61 байт
```

На проверенном Ballu ответ выглядит так:

```
5 байт заголовка + 59 байт тела + 1 байт контрольной суммы = 65 байт
```

Компонент адаптирован именно под второй вариант.

## Быстрая установка

### 1. Скопируйте компонент

Скопируйте папку:

```
components/tcl_climate
```

в каталог ESPHome:

```
/config/esphome/components/tcl_climate
```

### 2. Скопируйте конфигурацию

Скопируйте:

```
examples/ballu-tcl-65.yaml
```

например в:

```
/config/esphome/ballu-tcl-ac.yaml
```

### 3. Заполните secrets.yaml

Возьмите значения из:

```
examples/secrets.yaml.example
```

и добавьте собственные данные в:

```
/config/esphome/secrets.yaml
```

### 4. Проверьте пины

По умолчанию:

```yaml
uart_tx_pin: GPIO3
uart_rx_pin: GPIO1
```

Значения меняются в блоке `substitutions` в начале конфигурации.

### 5. Соберите и прошейте

В ESPHome:

```
Clean Build Files
Validate
Install
```

Для первой установки рекомендуется USB. После успешного запуска можно использовать OTA.

## Проверка после прошивки

Проверяйте функции по одной, выдерживая несколько секунд между тестами:

1. Включение и выключение.
2. `COOL`, температура 22 °C.
3. `HEAT`, температура 24 °C.
4. `FAN_ONLY`.
5. `DRY`.
6. Вертикальное качание по всему диапазону.
7. Вертикальное качание только в верхней части.
8. Фиксация вертикальных жалюзи по центру.
9. Отключение звукового подтверждения.
10. Выключение дисплея.

## Home Assistant

После добавления устройства появятся:

- одна сущность `climate`;
- две сущности `select` для жалюзи;
- два переключателя `switch` для звука и дисплея.

Стандартное окно `climate` умеет показать только:

- выключено;
- вертикально;
- горизонтально;
- оба направления.

Расширенные положения жалюзи доступны через отдельные сущности `select`.

Пример карточки находится в:

```
examples/lovelace-card.yaml
```

## Сеть и безопасность

В примере нет:

- реального Wi-Fi SSID;
- пароля Wi-Fi;
- API-ключа;
- OTA-пароля;
- статического IP;
- персонального имени устройства.

Рекомендуется выдавать постоянный адрес через DHCP reservation на роутере, а не вшивать IP в публичный YAML.

## Важное предупреждение по электрической части

На некоторых платах TX кондиционера работает уровнем около 5 В. GPIO ESP32-C3 рассчитаны на 3,3 В.

Не оставляйте 5 В напрямую на RX ESP32. Используйте согласование уровней, описанное в `docs/WIRING.md`.
